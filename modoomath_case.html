02 PROBLEM

한 화면에 모든 기능이 쏟아져,
무엇부터 해야 할지 알 수 없었습니다.

기존 메인 한 장에 서비스 소개·문제 생성·학습 자료·Q&A가 동시에 등장합니다. 사용자가 가장 먼저 마주하는 화면을 분석해 이탈을 부르는 4가지 문제를 정의했습니다.

P1

정체성 불명확

프린트·바로풀기 모드를 바꿔도 메인이 그대로라, 무엇을 하는 서비스인지 한눈에 와닿지 않는다.

P2

기능 과다 노출

메인에 모든 기능이 한 번에 쏟아져, 무엇부터 해야 할지 몰라 이탈로 이어진다.

P3

정보 위계질서 붕괴

제목·기능·안내가 모두 같은 비중으로 놓여, 무엇이 먼저인지 시선의 우선순위가 사라진다.

P4

중복 이름 버튼 과다

"바로풀기"가 서로 다른 페이지로 향하는데 이름이 같고, 풀이 중에도 "선택" 버튼이 남아 시선이 분산된다.

03 BENCHMARK

잘 만든 학습 서비스는, 명확했다.

방향을 잡기 위해 학습 콘텐츠 서비스의 주요 화면을 살펴봤습니다.
학습 유도를 위하여 기능을 늘어놓지 않고, '지금 할 일' 하나를 가장 앞에 둡니다.

reference · 가치 제안형
REF 01 가치 제안형

한 줄의 가치 메시지와 단일 CTA로 시작하여 군더더기 없이
핵심 행동 하나로 사용자를 안내하는 히어로 페이지를 구성했다.

reference · 콘텐츠 탐색형
REF 02 콘텐츠 탐색형

상단에 검색·핵심 액션을 크게 배치하고, 좌측에 학습 관련 콘텐츠를 일목요연하게 배치해 학습 시 '무엇을 할 수 있는지'가 한눈에 들어온다.

Takeaway

모두매쓰의 수많은 기능을 분리해 명확한 사용자 과정을 만들어야 한다.

04 KEY INSIGHTS

기능보다 "무엇을 할지"가 먼저였다.

01

사용자는 기능 목록보다
"지금 무엇을 해야 하는가"를 먼저 찾는다.

02

좋은 화면은 기능을 나열하지 않고
핵심 행동 하나를 가장 먼저 보여준다.

03

학습 서비스의 완주는 결국
신뢰의 흐름에서 시작된다.

12 RETROSPECTIVE
"더하는 것보다 무엇을 덜어낼지 정하는 일이 더 어렵고, 더 중요했다."

좋은 UX는 기능을 나열하는 게 아니라, 사용자가 목표에 가장 빨리 닿도록 돕는 것이었습니다. 모두매쓰의 메인은 모든 걸 보여주려다, 정작 아무것도 안내하지 못하고 있었습니다.

이번 리뉴얼에서 가장 큰 결정은 '추가'가 아니라 '제거'였습니다 — 중복 버튼을 지우고, 핵심 행동 하나를 가장 앞에 두는 것. 사용자를 위한 디자인은 결국 선택과 절제의 문제임을 배웠습니다.
