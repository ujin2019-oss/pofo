02 PROBLEM DISCOVERY

한 화면에 모든 기능이 쏟아져,
무엇부터 해야 할지 알 수 없었습니다.

기획서 분석 단계에서 사용자 이탈을 유발하는 4가지 핵심 문제를 정의했습니다. (출처: 모두매쓰 UX 개선 계획서 — 박효은·장수빈·유진, 2026.04.06)

P1

메인에 모든 기능을 한 번에 노출

무엇부터 할지 몰라 이탈 가능성↑, 탐색이 길어 피로도↑.

P2

서비스 설명 부족·정체성 불명확

프린트/바로풀기 모드를 바꿔도 메인 화면이 그대로라 사용법이 혼란스럽다.

P3

문제풀기 내 중복 이름 버튼 과다

"바로풀기"가 서로 다른 페이지로 가는데 같은 이름, "선택" 버튼이 풀이 중에도 남아 시선이 분산된다.

P4

정답 입력 버튼을 찾기 어려움

계산기·낙서 기능에 가려지고, 작은 햄버거 바를 눌러야 정답 입력란이 보인다.

BEFORE

기존 메인 화면

  • 1 기능 과다 노출 — 메인에 모든 기능이 한 번에 표시됨
  • 2 정체성 불명확 — 모드를 바꿔도 메인 변화 없음
  • 3 중복 버튼 — 같은 이름의 버튼이 다른 동작
  • 4 정답 입력 숨김 — 햄버거 바 뒤에 가려짐
03 PROJECT GOAL

혼란을 줄이고, 핵심에 집중하도록.

"메인에서 서비스 정체성을 알 수 있도록, 핵심 기능에 충실한 사이트로."

— 개선 방향
01

명확한 메인 구조

한눈에 들어오는 레이아웃·카피·CTA로 혼란스러운 메인을 정리한다.

해결: 무엇부터 할지 모름 · P1·P2
02

최단 경로 여정 설계

핵심 과업까지의 동선을 짧게 재설계해 긴 사용 과정을 단축한다.

해결: 긴 사용 과정 · P3
03

버튼 다이어트

기능 없는 버튼은 지우고 핵심 버튼의 가독성을 높인다.

해결: 핵심 기능 불명확 · P3·P4
04 KEY INSIGHTS

기능보다 "무엇을 할지"가 먼저였다.

01

사용자는 기능 목록보다 "지금 무엇을 해야 하는가"를 먼저 찾는다. 메인은 기능 나열이 아니라 한 가지 길을 제시해야 한다.

02

좋은 화면은 기능을 나열하지 않고 핵심 행동(Core Action) 하나를 가장 먼저 보여준다.

03

학습 서비스의 완주는 결국 신뢰의 흐름(Trust-flow)에서 시작된다.

05 USER FLOW

11 RETROSPECTIVE
"더하는 것보다 무엇을 덜어낼지 정하는 일이 더 어렵고, 더 중요했다."

좋은 UX는 기능을 나열하는 게 아니라, 사용자가 목표에 가장 빨리 닿도록 돕는 것이었습니다. 모두매쓰의 메인은 모든 걸 보여주려다 아무것도 안내하지 못하고 있었습니다.

이번 리뉴얼에서 가장 큰 결정은 '추가'가 아니라 '제거'였습니다 — 중복 버튼을 지우고, 핵심 행동 하나를 가장 앞에 두는 것. 사용자를 위한 디자인은 결국 선택과 절제의 문제임을 배웠습니다.
